## 一、产品背景与目标

### 1.1 现状

当前库存管理首页（`/inventory`）为**单一扁平表格视图**，5 列（食材名称 / 一级分类 / 二级分类 / 当前库存量 / 计量单位），全部食材平铺展示。页面无全局库存指标、无分类聚合、无库存健康度提示。用户进入后需逐行扫表才能感知整体库存状态，**无法一屏看清各类别食材的库存概况**。

### 1.2 目标

重构为**「分类聚合 + 卡片视图 + 指标看板」**的复合视图，达成以下目标：

- 一屏可见：全局库存关键指标（当前库存食材种类、低库存预警、临期食材预警、今日更新）
- 一屏可见：各一级分类的食材品种概览（仅品种数）
- 分类即入口：点击分类后下钻该分类的食材明细卡片
- 卡片突出库存量：大数字 + 健康度颜色指示，快速识别异常项；悬浮即见食材全字段详情

### 1.3 产品定位

库存管理首页是**高频查看页面**（每日多次），目标用户为后厨负责人 / 采购员。核心定位是「**库存状态仪表盘**」，不是操作页（入库/出库在采购/排程模块完成）。因此首页以**可读性、快速扫视**为第一优先级。

## 二、用户角色与场景

| 角色 | 使用频率 | 核心场景 | 核心需求 |
|------|----------|----------|----------|
| **后厨负责人** | 每日 3-5 次 | 晨会前扫一遍库存，判断今日采购重点 | 一眼看出哪些类别缺货、哪些品种预警、哪些临期 |
| **采购员** | 每日 1-2 次 | 收到采购计划后，先查库存再下单 | 快速定位某个食材的当前库存量，悬浮看详情确认规格/单位 |
| **管理员（店长）** | 每周 1-2 次 | 周盘点后检查库存健康度 | 看全局指标：总品种、预警数、临期数、更新活跃度 |

**三态分布**：
- 日常态（80%）：打开页面 → 看 KPI → 扫分类概览 → 点击有预警的分类 → 查看具体食材 → 悬浮看详情 → 决定采购
- 盘点态（15%）：按分类逐个核对，使用列表视图（表格）逐条确认
- 搜索态（5%）：已知某个食材名称，直接搜索 → 查看库存量 → 悬浮看详情

## 三、核心痛点与价值主张

### 3.1 痛点清单（双因素矩阵）

| 痛点 | 影响人数 | 频率 | 优先级 |
|------|----------|------|--------|
| 无法一屏看全局，需要逐行扫表 | 后厨负责人 + 采购员 + 店长 | 高频 | 🔴 优先 |
| 无库存健康度标识，无法快速识别异常 | 后厨负责人 + 采购员 | 高频 | 🔴 优先 |
| 食材详情需要跳页查看，打断扫货节奏 | 采购员 | 中频 | 🔴 优先 |
| 分类切换低效（仅下拉框筛选二级分类） | 后厨负责人 | 中频 | 🟡 第二 |
| 加载状态简陋（灰色脉冲块） | 所有用户 | 每次打开 | 🟡 第二 |
| 空状态无引导操作 | 新用户 / 初始化时 | 低频 | ⚫ 挂起 |

**只做 🔴 区**，其余挂起或纳入 Round 2/3。

### 3.2 价值主张

> 将「逐行扫表」的被动浏览模式，升级为「先看指标、再看分类、最后看明细」的主动扫视模式，让用户在 3 秒内完成全局库存健康度判断，在 10 秒内定位到需要关注的食材，悬浮即见全字段详情无需跳页。

**量化目标**：
- 用户定位到目标库存信息的路径：从「平均 5-8 次滚动/筛选」缩短到「1 次点击 + 局部扫视」
- 库存预警发现时间：从「主动扫表时偶然发现」变为「进入页面即看到 KPI 预警数」
- 查看食材详情：从「点击跳转到食材档案页」变为「悬浮卡片即见全字段」

### 4.2 MVP1 不做（挂起）

| 功能项 | 挂起原因 | 预期讨论时机 |
|--------|----------|-------------|
| 临期食材预警（真实数据） | 现有数据模型无保质期/过期日期字段，需先设计保质期管理子模块 | 保质期管理模块规划时 |
| 低库存阈值动态计算 | 本版写死 5.0，动态阈值需历史采购数据 + 算法设计 | Round 3 体验打磨时 |
| 分类颜色按食材类型着色 | 现有 `globals.css` tag-* 变量是硬编码十六进制，暗色模式对比度风险 | 暗色模式全局优化时 |
| 空状态引导操作（如"去采购"） | 需确认跳转链路（库存页 → 采购下单页的路径） | 采购模块整合优化时 |
| 食材卡片点击下钻到详情页 | 悬浮浮层已覆盖详情查看需求，点击下钻作为补充入口留到后续 | 食材档案模块扩展时 |
| 后端聚合 API（`/api/inventory/summary`） | 前端聚合几百条数据可接受，后端 API 非阻塞项 | 数据量超过 500 条或性能瓶颈时 |
| 分页/虚拟滚动 | 库存品种数通常 <200，无需分页 | 品种数超过 200 时 |
| 库存趋势图表（近 7 日变化） | 需历史库存快照数据 | MVP3 规划 | 本版不做 |

## 附录 A：数据定义与聚合规则

### A.1 数据流

```
现有 API: GET /api/inventory
  → 返回 InventoryItem[]（扁平数组，含 id, name, code, l2Code, l1Name, l2Name, currentQty, unit, updatedAt）
  → 前端 useMemo 聚合，无需后端改动

食材清单字段（悬浮浮层展示）：
  → 编码 code, 名称 name, 别名 alias, 采购规格 purchaseSpec,
  → 采购单位 purchaseUnit, 库存单位 stockUnit, 最新参考价 latestRefPrice,
  → 季节 season, 储存方式 storage, 二级分类 l2Name
```

### A.2 聚合规则

| 指标 | 计算规则 | 备注 |
|------|----------|------|
| 当前库存食材种类 | `rows.length`（currentQty > 0 的库存记录数） | 复用现有 API 的 `where: { currentQty: { gt: 0 } }` |
| 低库存预警数 | `rows.filter(r => r.currentQty < LOW_STOCK_THRESHOLD).length` | 阈值 `LOW_STOCK_THRESHOLD = 5.0`（本版写死） |
| 临期食材预警数 | `0`（或 `'—'`） | 依赖保质期数据模型，当前版本占位，待 MVP4 接入 |
| 今日更新数 | `rows.filter(r => isToday(new Date(r.updatedAt))).length` | 使用 `date-fns` `isToday` |
| 分类品种数 | 按 `l1Name` 分组后取 `group.length` | 未分类（`l1Name` 为空）归入「未分类」 |

### A.3 库存健康度规则

| 状态 | 条件 | 颜色指示 | 说明 |
|------|------|----------|------|
| 充足 | `currentQty >= 10` | `border-success`（绿色） | 日常库存充足 |
| 正常 | `5 <= currentQty < 10` | `border-warning`（黄色） | 库存正常，关注即可 |
| 低库存 | `currentQty < 5` | `border-destructive`（红色） | 需尽快采购补货 |

> **本版阈值写死**：充足 ≥10，正常 5-10，低库存 <5。单位为库存单位（各食材可能不同）。后续 MVP3 支持动态阈值。

## 附录 B：核心设计机制

### B.1 视觉风格（匹配系统全局）

严格复用项目 `globals.css` 的语义变量，不使用任何硬编码色值：

| 元素 | 颜色 Token | 说明 |
|------|-----------|------|

| 次要文字 | `text-muted-foreground`（oklch(0.556 0 0) / 灰色） | 描述、辅助信息 |

| 库存充足指示 | `border-success`（oklch(0.55 0.15 145) / 绿） | 左侧竖线 |

### B.3 食材卡片设计

```
┌──────────────────────────────┐
│  胡萝卜                     │  ← 食材名 (text-base font-medium)
│  ┌─────┐                    │
│  │ 12.5 │  kg              │  ← 库存量 (text-2xl font-bold) + 单位
│  └─────┘                    │
│  库存充足 │ 2天前更新       │  ← 健康度 badge + 更新时间 (text-xs text-muted-foreground)
└──────────────────────────────┘
```

- 卡片左侧 4px 竖线指示健康度（`border-l-4`）
- **不展示二级分类 badge**（二级分类在分组标题行已展示，避免重复）
- 库存量使用 `text-2xl font-bold`，视觉权重最高
- 更新时间使用 `date-fns` `formatDistanceToNow` 格式化为相对时间（如"2天前"）
- **悬浮浮层**：鼠标移入卡片后 200ms 延迟展示，移出后消失。浮层展示食材清单全字段：

```

│ 食材详情                            │
│ ─────────────────────────────────── │
│ 编码:        CAR-001                │
│ 名称:        胡萝卜                 │
│ 别名:        红萝卜                 │
│ 采购规格:    500g/袋                │
│ 采购单位:    袋                     │
│ 库存单位:    kg                     │
│ 最新参考价:  ¥3.50                  │
│ 季节:        四季                   │
│ 储存方式:    常温                   │
│ 二级分类:    根菜类                 │
└─────────────────────────────────────┘
```

### B.4 响应式断点
