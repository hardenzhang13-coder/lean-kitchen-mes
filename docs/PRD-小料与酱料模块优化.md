## 一、产品背景与目标

### 1.1 背景

菜品档案是精益厨房一切业务的起点。当前菜品模块处于半成品状态，核心阻塞在于：**小料和酱料的数据结构无法被菜品 BOM 引用**。

- 小料表（`minor_ingredients`）独立存在且数据为 0 条，字段与净料表高度重叠，却需单独维护，运营人员未使用。
- 酱料表（`sauce_ingredients`）缺少「类型」「规格」字段，无法区分自制酱料与外购酱料，菜品 BOM 引用时信息不完整。

### 1.2 目标

本次优化完成两件事：

1. **小料**：废弃独立表，数据统一存储在净料表（`net_ingredients`），通过食材分类体系区分。小料清单页保持独立 CRUD 体验。
2. **酱料**：精简并补全字段，使其具备菜品 BOM 引用的完整信息。

### 1.3 价值主张

- **合规（数据基础）**：让菜品 BOM 能完整引用 5 类食材（原料/净料/小料/调料/酱料），使菜品档案从"半成品"变为"可闭环"。
- **降本**：小料不再独立维护，减少一张表、一套 API、一个页面表单的心智负担。

## 二、用户角色与场景

| 角色 | 核心任务 | 使用页面 |
|------|----------|----------|
| 运营人员 | 维护食材基础数据，确保菜品 BOM 可选食材完整 | `/ingredients/minor`、 `/ingredients/sauce` |

### 核心场景

1. **维护小料**：运营人员在"小料清单"新增蒜末、姜片等，填写规格、单价，系统自动归入小料分类。
2. **维护酱料**：运营人员在"酱料清单"新增豆瓣酱，标注类型（自制/外购）、配方、品牌，单位默认克。
3. **（未来）配置菜品 BOM**：在菜品详情页选择小料/酱料，填入用量(g)即可。

## 四、产品范围（MVP1）

### 4.1 小料清单页（`/ingredients/minor`）

**功能项与规则：**

| 功能项 | 规则 | 验收标准 |
|--------|------|----------|
| 列表展示 | 读取 `net_ingredients` 中一级分类为"小料"的数据，表格字段：编号、名称、原料来源、规格、单价、单位、二级分类 | 展示数据与净料表一致，无重复条目 |
| 新增小料 | 弹窗表单，字段参照净料表单；**二级分类自动限定为"小料"下选项**；原料来源、出成率非必填；其余必填 | 提交后数据落入 `net_ingredients`，`l1_code` 关联小料分类 |
| 编辑小料 | 同新增；编号只读 | 修改后列表即时刷新 |
| 删除小料 | 二次确认 Modal；直接删除（当前无菜品引用） | 删除后数据从 `net_ingredients` 移除 |
| 分页 | 默认 20 行/页，统一分页组件 | 与原料/净料列表分页规范一致 |

### 4.2 酱料清单页（`/ingredients/sauce`）

**功能项与规则：**

| 功能项 | 规则 | 验收标准 |
|--------|------|----------|
| 列表展示 | 读取 `sauce_ingredients`，表格字段：编号、名称、商品/品牌名称、规格、单价、单位、类型 | 类型字段展示为 badge（自制/外购） |
| 新增酱料 | 弹窗表单：名称（必填）、商品/品牌名称（必填）、规格（必填）、单价（必填）、单位（固定为"g"，只读展示）、配方（多行文本，必填）、类型（TileGroup 平铺：自制/外购，必填） | 提交后 `unit` 默认写入 'g'，无需选择 |
| 编辑酱料 | 同新增；编号只读 | 修改后列表即时刷新 |
| 删除酱料 | 二次确认 Modal | 直接删除（无菜品引用） |
| 分页 | 默认 20 行/页 | 统一分页规范 |

### 4.3 关联变更

| 变更点 | 说明 |
|--------|------|
| 废弃 `/api/minor-ingredients` | 前端小料页改为调用 `/api/net-ingredients?category=小料`（或等效筛选参数） |
| 废弃 `minor_ingredients` 表 | Schema 中删除（无数据，无迁移风险） |
| 净料页面过滤 | `/ingredients/net` 查询时过滤掉小料分类数据，避免重复展示 |
| `dish_minor_details` 外键切换 | 指向 `net_ingredients.id`（当前无数据，直接改 Schema） |

## 附录 A：数据模型变更

### A.1 废弃表

| 表名 | 动作 | 原因 |
|------|------|------|
| `minor_ingredients` | 删除 | 数据为 0，字段与净料重叠，统一走 `net_ingredients` |
| `/api/minor-ingredients` | 删除 | 前端改为调用 `/api/net-ingredients` 带分类筛选 |

### A.2 `net_ingredients` 调整

| 字段 | 变更 | 说明 |
|------|------|------|
| `source_ingredient_id` | 改为 `Int?`（Optional） | 小料无原料来源，允许为空 |
| `yield_rate` | 改为 `Decimal?`（Optional） | 小料无出成率概念，允许为空 |
| `l2_code` | 保持必填 | 小料必须选择小料分类下的二级分类 |

### A.3 `sauce_ingredients` 调整

| 字段 | 类型 | 变更 | 说明 |
|------|------|------|------|
| `brand` | `varchar(100)` | ➕ 新增 | 商品/品牌名称，如"郫县豆瓣酱" |
| `spec` | `varchar(100)` | ➕ 新增 | 规格，如"500g/瓶" |
| `type` | `varchar(20)` | ➕ 新增 | 枚举：自制 / 外购 |
| `storage` | `varchar(10)` | ➖ 移除 | 不再维护储存方式 |
| `unit` | `varchar(20)` | 默认值 'g' | 前端只读展示，不开放选择 |
| `unit_price` | `decimal(10,2)` | 保持 | 单价 |
| `recipe` | `text` | 保持 | 配方说明，多行文本 |
| `name` | `varchar(100)` | 保持 | 酱料名称 |
| `code` | `varchar(20)` | 保持 | 编号 SAU-0001，服务端自动生成 |

### A.4 API 调整

| API | 变更 | 说明 |
|------|------|------|
| `GET /api/net-ingredients` | 新增 `?l1Code=MIN` 或 `?category=小料` 筛选 | 供小料列表页调用 |
| `POST /api/net-ingredients` | 新增时如来自小料页面，自动校验 `l2_code` 属于小料分类 | 后端校验，防止错分类 |
| `GET /api/sauce-ingredients` | 返回字段增加 `brand`/`spec`/`type`，移除 `storage` | 列表/详情同步 |
| `POST/PUT /api/sauce-ingredients` | 接收新字段，移除 `storage`；`unit` 默认 'g' | 表单提交兼容 |

## 附录 B：核心设计机制 — 小料数据源切换方案

### B.1 为什么小料走净料表

小料与净料的数据结构高度一致：都有名称、规格、单价、单位、分类、储存方式。差异仅在于：
- 净料有"原料来源"和"出成率"（小料没有，改为非必填即可）
- 小料有独立分类体系（分类字段本身就在净料表中）

独立表导致：一套字段、一套 API、一个页面，却无人维护（数据 0 条）。统一后，小料作为净料的一个分类子集，运营人员在"小料清单"页面看到的仍是独立视图，但底层数据一致。

### B.2 分类锁定机制

小料页面的二级分类选择器，只查询 `ingredient_categories_l2` 中 `parent_code` 等于小料一级分类（如 `MIN`）的选项。新增/编辑时，后端再次校验 `l2_code` 的 `parent_code` 是否属于小料分类，防止前端绕过。

### B.3 净料页面过滤

`/ingredients/net` 调用 `GET /api/net-ingredients` 时，默认排除小料分类的数据。实现方式：后端查询时加 `NOT l2_code IN (小料分类下的所有二级分类)`。

## 附录 C：术语表

| 术语 | 定义 | 英文 |
|------|------|------|
| 小料 | 用量极小、起增香/去腥/提味作用的食材，如蒜末、姜片、葱花 | Minor Ingredient |
| 酱料 | 复合加工调味半成品，决定菜品味型方向，如豆瓣酱、辣椒油 | Sauce Ingredient |
| 净料 | 原料经初加工后的规格化半成品。本次小料归入净料表，通过分类区分 | Net Ingredient |
| 原料来源 | 该净料由哪种原料加工而来。小料场景无来源，字段非必填 | Source Ingredient |
| 出成率 | 原料加工为净料后的产出比例。小料场景无此概念，字段非必填 | Yield Rate |
