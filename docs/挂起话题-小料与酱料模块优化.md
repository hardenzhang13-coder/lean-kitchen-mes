# 挂起话题清单

> 来源：PRD「小料与酱料模块优化」v0.1
> 规则：每条挂起项必须带"预期讨论时机"，不遗漏、不口头说说。

---

## 1. 菜品 BOM 引用小料与酱料

- **内容**：菜品模块启动后，`dish_minor_details` 和 `dish_sauce_details` 需要支持从优化后的小料/酱料数据源选择食材。小料选择器从 `net_ingredients` 筛选，酱料选择器从 `sauce_ingredients` 读取。
- **预期讨论时机**：菜品模块开发启动时（MVP2），在菜品 BOM 设计阶段一并确定。
- **当前状态**：⏸️ 挂起

## 2. 小料二级分类细分

- **内容**：当前小料一级分类下的二级分类是否足够？如是否需要细分"姜蒜类"、"香料类"、"葱花类"等。运营人员尚未提供完整分类清单。
- **预期讨论时机**：小料数据录入超过 20 条时，或运营人员提出查找困难时，回来看是否需要细分。
- **当前状态**：⏸️ 挂起

## 3. 净料页面是否展示小料

- **内容**：当前方案是净料页面过滤小料，但未来如果运营人员认为"小料也是净料的一种，应该在一起看"，需要重新评估。
- **预期讨论时机**：运营人员使用 2 周后，收集反馈再决定。
- **当前状态**：⏸️ 挂起

## 4. 小料/酱料批量导入

- **内容**：当前原料已有 CSV 批量导入功能。小料（走净料表）理论上可复用原料导入逻辑，但分类需要自动识别。酱料批量导入待评估。
- **预期讨论时机**：小料/酱料数据量超过 50 条时，评估批量导入 ROI。
- **当前状态**：⏸️ 挂起

## 5. 酱料配方字段的用途

- **内容**：`recipe`（配方）当前定义为"多行文本输入"，但未来菜品 BOM 引用酱料时，是否需要解析配方中的成分？当前仅作为展示字段，未做结构化。
- **预期讨论时机**：菜品模块需要"根据酱料配方自动计算营养成分/过敏原"时，再评估是否将配方拆分为结构化子表。
- **当前状态**：⏸️ 挂起
