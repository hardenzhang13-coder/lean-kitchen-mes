# 菜品档案模块优化

> v0.1 初稿

---

## 一、产品背景与目标

菜品档案是排程、采购、库存全链路的起点。当前为半成品：BOM允许人工输入非食材库名称，数据脱节、成本无法闭环；状态机三态（草稿/待处理/已发布）语义混乱；创建在弹窗内分5步，BOM与工艺无法一览全貌。

**目标**：实现菜品档案CRUD完整闭环——BOM从食材库强制引用，状态两态清晰，创建/编辑在独立页面一览完成，列表用表格高效查找。

**价值**：合规。让菜品BOM成为全链路的可信数据源，消除人工估算误差。

---

## 二、用户角色与场景

**运营人员**：维护菜品档案，确保BOM完整、成本准确。

**核心场景**：新增"宫保鸡丁"，从食材库选择主料、辅料、小料、调料，系统自动带出单价，填入用量后估算成本；已发布菜品发现主料用量需调整，撤回为草稿，修改后重新发布。

---

## 三、核心痛点与价值主张

| 痛点 | 优化后价值 |
|------|------------|
| BOM允许人工输入食材名称，成本无法计算 | 强制从食材库选择，成本自动闭环 |
| 状态机三态语义混乱，"待处理"无业务逻辑 | 两态清晰：草稿可改，已发布锁定 |
| 创建在弹窗内分5步，BOM与工艺割裂 | 独立页面左右一览，信息不割裂 |
| 列表卡片信息密度低，一行4个查找困难 | 表格一行即全貌，筛选排序高效 |

## 四、产品范围（MVP1）

### 4.1 列表页（表格）

字段顺序：序号、编号、菜品名称、类别、菜品描述、荤素、成本价、状态、创建时间。类别与状态使用浅色胶囊（Badge）展示。描述字段超过宽度后内容用"…"截断显示。操作按钮常驻：查看+编辑。删除和撤回操作移至详情页。点击行（操作区除外）右侧滑出弹窗，占屏幕宽度40%。点击编辑跳转独立编辑页。

### 4.2 侧边弹窗详情（占40%）

展示三区块：基础信息（编号、类别、菜系、做法、口味、份量、季节、荤素、描述）+ BOM汇总（各类食材数量+总成本）+ 工艺预览（4阶段各步骤数量）。底部操作：草稿态显示删除按钮，已发布态显示下架按钮（点击后确认，状态变为草稿）。

### 4.3 创建/编辑页（独立页面，左右4:6）

**左侧（40%）：基础信息**

字段：名称、类别、描述、菜系、做法、口味、荤素类型、份量、季节，全部在一个信息块内。标签和输入文字统一为小字号，placeholder更小两级；placeholder超过5字统一为"请选择"或"请输入"。选项≤5时横向平铺，字号比标签小两级；平铺选项必须展示在一行，若一行展示不完整，则该字段单独一行（独占整行）展示。描述并入基础信息块，不独立成块。

**右侧（60%）：菜品用料（BOM）**

5类食材（主料、辅料、小料、调料、酱料）纵向堆叠，不是Tab。必须从食材库选择：主料/辅料/小料从净料表，调料从调料表，酱料从酱料表；禁止手动输入。选择弹窗960px宽，一二级分类默认选中第一项，分类区2列平铺，列表展示编号、名称、食材名称/品牌、规格、单价、单位。

已选项展示：名称、用量（克）、单位（克）、成本价（实时计算）。小料/调料/酱料额外显示商品名称/品牌；主料/辅料不显示。酱料可空，其他4类为发布必填；空时显示"暂无酱料（可选）"。页面实时显示估算总成本，修改用量后即时更新。

**右侧（60%）：加工工艺**

4阶段（初加工、预处理、上灶加工、出锅成品）纵向排列，左侧虚线连接形成顺序逻辑线。点击阶段标题可展开/收起，新创建时全部展开。编辑某步骤时，该步骤展开为表单，同阶段其他步骤折叠为预览行，其他阶段完全收起（仅保留标题）。

步骤表单3行：第1行对象（从已添加的主料辅料中选择）+动作；第2行描述（多行文本，独占一行）；第3行工具+标准（标准限50字）。步骤预览：每步显示编号、对象→动作、描述（单行截断）、编辑/删除/上移/下移按钮。

### 4.4 状态机与发布规则

**状态**：草稿、已发布。废弃"待处理"。

**发布校验**：基本信息全部必填 + 主料≥1 + 辅料≥1 + 小料≥1 + 调料≥1 + 工艺≥1步。酱料可空。任一不满足，提示具体缺失项。

**已发布锁定**：不可修改BOM、不可修改工艺、不可删除。只能下架为草稿，下架后数据保留，可继续编辑。发布时后端自动计算成本。成本计算在用量输入框光标移出（blur）后触发，计算务必准确。

## 附录 A：状态机定义

| 状态 | 语义 | 可执行操作 | 可迁移至 |
|------|------|------------|----------|
| 草稿 | 可编辑、可删除、不在排程中展示 | 编辑、删除、发布 | 已发布 |
| 已发布 | 锁定不可改、可在排程中选择 | 下架（变为草稿）、查看 | 草稿 |

**迁移触发**：草稿→已发布需通过全部必填校验；已发布→草稿需确认下架，BOM和工艺数据保留。

**操作权限**：草稿可改BOM/工艺/删除；已发布仅可查看和下架。

## 附录 B：核心设计机制

### B.1 BOM食材来源

| BOM类型 | 来源表 | 备注 |
|---------|--------|------|
| 主料 | 净料表（排除小料分类） | 用量必填 |
| 辅料 | 净料表（排除小料分类） | 用量必填 |
| 小料 | 净料表（小料分类） | 用量必填 |
| 调料 | 调料表 | 用量必填 |
| 酱料 | 酱料表 | 用量可选 |

### B.2 平铺选择器

- 选项≤5时：横向平铺，字号比标签小两级。
- 若一行放不下：该字段独占一行。
- 选项>5时：改用弹窗选择器。

### B.3 步骤编辑折叠

- 编辑某步时：该步展开为表单，同阶段其他步折叠为预览行，其他阶段完全收起（仅保留标题）。
- 新创建时：4阶段全部展开，显示空状态。
- 点击阶段标题：切换展开/收起。

## 附录 C：术语表

| 术语 | 定义 |
|------|------|
| BOM | 菜品用料清单，由主料、辅料、小料、调料、酱料5类组成 |
| 草稿 | 未发布状态，可编辑、可删除，不在排程中展示 |
| 已发布 | 锁定状态，可在排程中选择，成本已计算 |
| 下架 | 已发布→草稿的反向操作，BOM和工艺数据保留 |
| 4阶段 | 初加工、预处理、上灶加工、出锅成品，加工工艺的固定阶段 |
| 对象 | 加工步骤的操作对象，必须从已添加的主料和辅料中选择 |
| 标准 | 加工步骤的完成标准，限50字 |
